SDL3GPU WebGPU Backend

SDL3GPU is the new graphics API for SDL3. It is designed to be a modern, easy-to-use, and high-performance graphics API for C and C++ developers. SDL3GPU is built on top of Vulkan, Metal, and Direct3D 12. I am working on porting the Vulkan backend fucntionality over to WebGPU so that SDL3 can target the web as well. This will allow for SDL3 applications to be run on nearly any device with a web browser. This project is still in development while I am in school.

Tint-WASM

Tint-WASM is a reverse engineered *as light as can be* WebAssembly runtime for Google's Tint (WGSL) compiler. It also includes access to the entirety of SPIRV-Tools so we can also target SPVASM. Currently Tint-WASM supports SPIR-V, SPIR-V ASM, and WGSL cross-compilation. Tint-WASM can be used in two different ways: as a standalone static library being used from the ground up, or the WASM module with C++ API that can be used in the browser. I plan on using Tint-WASM to help further my development of the WebGPU backend for SDL3GPU (previously Nano). This will prove very useful for shader reflection which will be used for pipeline creation.

Nano: C WebGPU Runtime

Nano is a C library that allows for rapid development of WebGPU applications. It provides a simple API for creating and managing WebGPU resources, and is designed to be easy to use and understand. Nano is built on top of the WebGPU C API, and is designed to be compatible with the WebGPU specification. Nano is currently in development, and is not yet feature complete. At the moment, Nano can create and manage WebGPU devices, implement CImGui, and generate buffers, textures, and pipelines based on shader code.
